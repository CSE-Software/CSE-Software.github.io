Admins only

First of all, join our GitHub page at CSE Software. You need to email professor Michael Heroux for permission.

Overview about our web page

Our web page contains 8 main sections including:

  • Best practice highlight: contain best exercises for CSE Software
  • Blog: contain all blog posted by members
  • Fellow program: contain all program
  • Getting started: contain basic methodologies guide
  • Search for scientific topic: contain all resources for major CSE Software activities
  • Sign up for notification: community engagement by sign up our google group
  • Training opportunities: guide for study CSE Software
  • Upcoming events: contain new announcements and upcoming workshop, conference.

Our web page also contain some navigations including:

  • Home: comeback to homepage
  • Community: introduction for community engagement
  • Bibliography: CSE Software Annotated Bibliography
  • About us: brief information about us and our project

How to edit and contribute our web page?

Prerequisite: basic understanding on HTML and Markdown. After that, follow this instruction

. Then, take a look at "gulpfile.js" file to see what can be changed and how to regenerate web page after generating. Full instruction on specific editing come follow.
  1. Edit main page:
    • All 8 main sections are written in markdown language located on "_post" directory. In the markdown file, the text between two "---" decode the theme while the rest of the file decide the content. Only need refresh web page after changing stuff.
    • Main section banner are written in HTML and all located on "category" directory. Only need refresh web page after changing stuff.
    • "_config.yml" contains important setup for main page. After change, run "gulp jekyll-rebuild" and then "jekyll serve" to validate the change.
  2. Edit page navigation:
    • Update navigation content: Web page navigation locate in the correspond HTML files. Only need refresh web page after changing stuff.
    • Add/Remove navigation: Go to "_config.yml", find "links" and add/remove link, then add/remove correspond html file. After change, run "gulp jekyll-rebuild" and then "jekyll serve" to validate the change.
  3. Update web page theme:
    Web page theme are located on "src/styl" directory. After change, run "gulp jekyll-rebuild" and then "jekyll serve" to validate the change.
  4. Writing blog:
    Blog posts are written in HTML and located in blog directory. Take a look at my sample blog for creating a new one.
